A reproducible build of the **working** setup: Debian 13 privileged LXC on Proxmox, sharing
a single RTX 2080 Ti across multiple containers, streamed over Moonlight/Sunshine with
GPU video (NVENC), keyboard/mouse input, and audio.

> **Status:** working — video, input, and audio all confirmed.
> **Driver version:** NVIDIA `610.43.02` (host kernel module **and** container userspace
> must match exactly; installed from apt, not the `.run` installer).
> **Container:** CTID 112, hostname `desktop`, IP `10.127.10.112` on bridge `vmbr50`
> (NAS VLAN). Stream user: `tiller` (UID 1000).

---

## 0. Architecture & key principles

- The GPU driver **kernel module** lives on the **Proxmox host**. The container gets only the
  **userspace libraries** (same version) plus device nodes bind-mounted in.
- The LXC is **privileged** with `nesting=1`. The GPU is *shared*, not PCI-passed-through, so
  multiple LXCs can use the same card at once.
- The streaming session is a **headless Xorg + XFCE** session started by the
  `desktop@tiller.service` system unit running `startx` (no display manager — `seat0` is not
  graphical with the NVIDIA driver). Sunshine is launched from `.xinitrc`.
- Audio works only when the session has a real **systemd user session**
  (`XDG_RUNTIME_DIR` + D-Bus) so it can reach PipeWire — achieved with
  `loginctl enable-linger`, not the brittle `PAMName=login`.

## 2. Create the LXC container

```bash
pct create 112 local:vztmpl/debian-13-standard_*_amd64.tar.zst \
  --hostname desktop \
  --cores 16 --memory 32768 --swap 16384 \
  --rootfs local-nvme2-thin:250 \
  --unprivileged 0 \
  --features nesting=1 \
  --net0 name=eth0,bridge=vmbr50,firewall=1,gw=10.127.10.1,ip=10.127.10.112/24,type=veth \
  --nameserver 10.127.10.1 \
  --onboot 1 --ostype debian
```

### 2.1 Final container config — `/etc/pve/lxc/112.conf`

```ini
arch: amd64
cores: 16
features: nesting=1
hostname: desktop
memory: 32768
nameserver: 10.127.10.1
net0: name=eth0,bridge=vmbr50,firewall=1,gw=10.127.10.1,hwaddr=BC:24:11:B3:5E:42,ip=10.127.10.112/24,type=veth
onboot: 1
ostype: debian
rootfs: local-nvme2-thin:vm-112-disk-0,size=250G
swap: 16384

# ---- GPU + input device access (verify majors against /proc/devices!) ----
lxc.cgroup2.devices.allow: c 13:* rwm      # /dev/input
lxc.cgroup2.devices.allow: c 10:223 rwm    # /dev/uinput
lxc.cgroup2.devices.allow: c 195:* rwm     # nvidia*
lxc.cgroup2.devices.allow: c 226:* rwm     # /dev/dri (DRM)
lxc.cgroup2.devices.allow: c 506:* rwm     # nvidia-uvm
lxc.cgroup2.devices.allow: c 509:* rwm     # nvidia-caps

# ---- proc/sys/cgroup rw (nesting=1 only gives sys:mixed; udev needs sys:rw) ----
lxc.mount.auto: proc:rw sys:rw cgroup:rw

# ---- bind the host device nodes in ----
lxc.mount.entry: /dev/input          dev/input          none bind,create=dir,optional 0 0
lxc.mount.entry: /dev/uinput         dev/uinput         none bind,optional,create=file
lxc.mount.entry: /dev/dri            dev/dri            none bind,optional,create=dir
lxc.mount.entry: /dev/nvidia0        dev/nvidia0        none bind,optional,create=file
lxc.mount.entry: /dev/nvidiactl      dev/nvidiactl      none bind,optional,create=file
lxc.mount.entry: /dev/nvidia-modeset dev/nvidia-modeset none bind,optional,create=file
lxc.mount.entry: /dev/nvidia-uvm     dev/nvidia-uvm     none bind,optional,create=file
lxc.mount.entry: /dev/nvidia-uvm-tools dev/nvidia-uvm-tools none bind,optional,create=file
lxc.mount.entry: /dev/nvidia-caps/nvidia-cap1 dev/nvidia-caps/nvidia-cap1 none bind,optional,create=file
lxc.mount.entry: /dev/nvidia-caps/nvidia-cap2 dev/nvidia-caps/nvidia-cap2 none bind,optional,create=file
```

> The cgroup-allow + mount-entry block can be copied into other LXCs to share the same card.
> **A cold `pct stop 112 && pct start 112` is required** for new bind mounts to take (a reboot
> from inside the container is not enough).

```bash
pct stop 112 && pct start 112
pct enter 112
```

## 5. The headless desktop service

`/etc/systemd/system/desktop@.service`:

```ini
[Unit]
Description=Headless X desktop for %i
After=systemd-user-sessions.service

[Service]
User=%i
# PAMName=login          # <-- intentionally DISABLED (see §9.1)
# Make DRM + uinput writable before the session starts:
ExecStartPre=+/bin/sh -c 'chmod 666 /dev/dri/card1 /dev/dri/renderD128 /dev/uinput || true'
# VLAN/UDP fragmentation workaround for the video stream (see §7):
ExecStartPre=+/usr/sbin/ethtool -K eth0 gso off gro off tso off tx-udp-segmentation off
Environment=XDG_SESSION_TYPE=x11
ExecStart=/usr/bin/startx -- :0 -keeptty -novtswitch
Restart=always
TimeoutStopSec=10
KillMode=mixed

[Install]
WantedBy=multi-user.target
```

`~tiller/.xinitrc` (owned by `tiller`, `chmod +x`):

```sh
#!/bin/sh
# Join the real user session so PipeWire/Pulse + systemd --user are reachable (audio!):
export XDG_RUNTIME_DIR="/run/user/$(id -u)"
export DBUS_SESSION_BUS_ADDRESS="unix:path=${XDG_RUNTIME_DIR}/bus"

# Wait for the pipewire-pulse socket before launching Sunshine:
for i in $(seq 1 20); do
    [ -S "${XDG_RUNTIME_DIR}/pulse/native" ] && break
    sleep 0.5
done

sunshine &
exec xfce4-session
```

Enable:

```bash
systemctl daemon-reload
systemctl disable --now lightdm        # ensure no DM fights for :0
systemctl enable --now desktop@tiller
```

Verify: `systemctl is-enabled desktop@tiller lightdm` → `enabled` / `disabled`, and
`loginctl` should show one `tiller` session of CLASS `manager` (the lingering user manager).

## 6. Audio (PipeWire)

### 6.1 Permanent user session (creates `/run/user/1000` + PipeWire)

```bash
loginctl enable-linger tiller
ls -la /run/user/1000        # must exist
sudo -u tiller XDG_RUNTIME_DIR=/run/user/1000 systemctl --user status pipewire-pulse
```

### 6.2 Virtual sink for Sunshine to capture (no physical sound card)

`~tiller/.config/pipewire/pipewire-pulse.conf.d/10-virtual-sink.conf`:

```
pulse.cmd = [
  {
    cmd = "load-module"
    args = "module-null-sink sink_name=virtual_speaker sink_properties=device.description=Virtual Speaker"
  }
]
```

Apply and make default:

```bash
systemctl --user restart pipewire pipewire-pulse wireplumber
wpctl status
wpctl set-default <virtual_speaker id>
```

### 6.3 Pin the sink in Sunshine (avoids the `auto_null` race)

At session start Sunshine may briefly fall back to `auto_null` ("Couldn't set
default-sink [auto_null]: No such entity") before `virtual_speaker` is up. Pin it in
Sunshine's config to remove the race:

```
audio_sink = virtual_speaker
```

(Find the exact name with `pactl list short sinks` if needed.)

## 7. Networking: VLAN + UDP segmentation (no-video fix)

Moonlight connects but shows no/black video over the VLAN because NIC **UDP segmentation
offload** mangles the large UDP video packets. The specific culprit flag was
`tx-udp-segmentation`. Fix:

```bash
ethtool -K eth0 gso off gro off tso off tx-udp-segmentation off
```

This is wired into `desktop@.service` as an `ExecStartPre`. For something that survives cold
reboots independently of the desktop unit, use a dedicated oneshot instead/as well —
`/etc/systemd/system/nic-offload.service`:

```ini
[Unit]
Description=Disable NIC offloads for Sunshine UDP video
After=network-online.target
Wants=network-online.target

[Service]
Type=oneshot
ExecStart=/usr/sbin/ethtool -K eth0 gso off gro off tso off tx-udp-segmentation off

[Install]
WantedBy=multi-user.target
```

```bash
systemctl enable --now nic-offload.service
```

Sunshine ports to allow on the `vmbr50`/`net0` firewall (and OPNsense LAN→NAS rule):
- TCP **47984, 47989, 48010**
- UDP **47998–48000, 48010**

## 9. Why things are the way they are (troubleshooting notes)

### 9.1 `PAMName=login` is disabled
It runs setup through `/etc/pam.d/login`, which has `session required pam_loginuid.so`.
That writes the audit login UID to `/proc/self/loginuid` — needs the kernel audit subsystem +
`CAP_AUDIT_CONTROL`, unavailable even in a privileged LXC. Being `required`, its failure
aborts the session and the service won't start. We skip PAM and get the runtime dir via
`enable-linger`. (Removing it also helped the **input** path: Xorg runs as root and bypasses
the broken logind `TakeDevice`.)

### 9.2 Input devices (the hard one)
Getting keyboard/mouse required ALL of:
- `lxc.mount.auto: proc:rw sys:rw cgroup:rw` — `nesting=1` only gives `sys:mixed`, but
  `systemd-udevd` in the container needs `/sys` writable.
- `lxc.cgroup2.devices.allow: c 13:* rwm` (+ `10:223` for uinput).
- Bind-mount `/dev/input` from the host: the container `/dev` is **tmpfs, not devtmpfs**, so
  udev can't `mknod` the nodes. Input devices aren't namespaced, so the host devtmpfs nodes
  pass straight through the bind mount.
- A **cold `pct stop/start`** after editing `112.conf` (restart-from-inside isn't enough).

### 9.3 `systemctl --user` fails inside Moonlight but works over SSH
The graphical session lacked `XDG_RUNTIME_DIR`/`DBUS_SESSION_BUS_ADDRESS`. Fixed by the
`.xinitrc` exports (§5) + `enable-linger` (§6.1). Verify in a Moonlight terminal:
`echo $XDG_RUNTIME_DIR` → `/run/user/1000`.

### 9.4 `nvidia-smi` "Driver/library version mismatch"
Host module and container userspace versions differ. Match both to `610.43.02` and
`apt-mark hold` (see §3.1).

### 9.5 `/dev/nvidia-uvm` missing in the container
Create it on the **host** first (`nvidia-modprobe -u -c=0`); it must exist there to be
bind-mounted in. Re-check dynamic majors (506/509) after updates.

### 9.6 Harmless log noise
`Couldn't release NvFBC context` (teardown on client disconnect), `Pen passthrough eraser:
No such device` (no wacom), blueman/bluez errors (no bluetooth), `wireplumber ... libcamera
SPA plugin is missing` (no camera) — all ignorable.
